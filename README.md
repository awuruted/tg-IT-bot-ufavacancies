# Ufa IT Vacancies Bot

Telegram-бот, который собирает IT-вакансии по Уфе с hh.ru и отвечает на
вопросы пользователя, используя LLM (через Amvera).

## Структура

- `app.py` — точка входа: запускает бота, планировщик и health-check сервер.
- `parser.py` — сбор вакансий через открытый API hh.ru (ключ не нужен).
- `db.py` — хранение вакансий в SQLite.
- `llm.py` — обращение к LLM через Amvera.
- `amvera.yaml` — конфигурация деплоя.

## Переменные окружения (задаются в настройках приложения на Amvera)

| Переменная           | Обязательна | Описание                                              |
|-----------------------|:-----------:|--------------------------------------------------------|
| `TG_BOT_TOKEN`         | да          | Токен бота от @BotFather                               |
| `AMVERA_LLM_TOKEN`     | да          | Токен доступа к LLM из личного кабинета Amvera          |
| `AMVERA_LLM_API_URL`   | нет         | Эндпоинт LLM API (если отличается от значения по умолчанию) |
| `AMVERA_LLM_MODEL`     | нет         | Модель (по умолчанию `gpt-4.1`)                         |
| `AMVERA`               | нет         | Ставится автоматически Amvera, переключает путь БД на `/data` |

## Локальный запуск

```bash
pip install -r requirements.txt
export TG_BOT_TOKEN="твой_токен"
export AMVERA_LLM_TOKEN="твой_токен"
python app.py
```

## Запуск только парсера (без бота)

```bash
python parser.py
```

## Деплой на Amvera

1. Создай проект на amvera.ru, выбери тариф.
2. Пропиши переменные окружения из таблицы выше в настройках приложения.
3. Привяжи свой Git-репозиторий или используй выделенный репозиторий Amvera.
4. Запушь код:

```bash
git remote add amvera https://git.amvera.ru/ВАШ_ЮЗЕРНЕЙМ/ВАШ_ПРОЕКТ
git add .
git commit -m "Initial commit"
git push amvera master
```

5. Дождись статуса «Приложение запущено», проверь логи на наличие ошибок.

## Важные нюансы

- Файлы вне `/data` не сохраняются между пересборками — база вакансий
  лежит именно там при запуске на Amvera.
- hh.ru может ограничивать частоту запросов — в парсере уже есть задержка
  между страницами, при необходимости увеличь её.
- LLM-эндпоинт в `llm.py` — рабочий шаблон; сверь точный URL и формат
  запроса в личном кабинете Amvera в разделе LLM/AI, при расхождении
  поменяй `AMVERA_LLM_API_URL`.
- Бесплатный лимит LLM на Amvera ограничен (~20 000 токенов на тест) —
  для постоянной работы бота закладывай бюджет на оплату по токенам.
